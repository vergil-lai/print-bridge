# 测试打印校准页设计

## 背景

PrintBridge 当前已经预留 `POST /print/test`，但接口仍返回 `NOT_IMPLEMENTED`，设置页也没有可用的测试打印入口。V1 设计文档曾把“设置页能执行测试打印”列为验收项，因此下一步把它补成一个本机诊断能力。

这个功能属于 Tauri Agent，不属于浏览器 JSSDK。它验证本机默认打印机、默认纸张、PDF 生成和平台打印提交链路，不扩展 WebSocket 业务协议。

## 目标

- 使用 Agent 当前默认打印机和默认纸张生成一页测试校准 PDF。
- PDF 页面物理尺寸必须等于默认纸张尺寸。
- 校准页能帮助用户检查偏移、缩放、方向和边缘裁切。
- 成功语义与业务打印一致：只表示已提交到系统打印队列，不表示打印机已经完成出纸。

## 非目标

- 不调用 Windows 或 macOS 的系统测试页。
- 不新增 JSSDK API。
- 不把测试打印塞进 WebSocket `print` / `print_batch` 协议。
- 不做打印机驱动配置修改。
- 不做打印完成检测。

## 后端设计

继续使用已有 HTTP 入口：

```text
POST /print/test
```

后端新增一个很薄的测试打印服务，职责如下：

1. 从当前 `AgentConfig` 读取 `printing.default_printer`。
2. 从当前 `AgentConfig` 读取 `printing.default_paper`。
3. 校验纸张宽高为正数且有限。
4. 生成一页矢量 PDF 校准页。
5. 调用现有 `PrintBackend::print_pdf()` 提交到系统打印队列，份数固定为 `1`。
6. 写入本地日志。

测试打印不进入业务打印队列，不分配 `job_id`，也不广播 WebSocket job 状态。它是本机诊断动作，不是 ERP 下发的业务任务。

## 校准页内容

校准页按默认纸张尺寸生成，例如默认纸张是 `60 x 40 mm`，PDF 页面也必须是 `60 x 40 mm`。

页面固定包含：

- 外边框：贴近页面边缘，用于检查裁切。
- 内边框：距页面边缘约 `2mm`，用于检查缩放和安全边距。
- 中心十字线：水平线和垂直线穿过页面中心，用于检查偏移。
- 四角定位标记：用于检查方向、旋转和边缘裁切。
- 尺寸标记：显示当前纸张尺寸，例如 `60 x 40 mm`。
- 诊断文字：`PrintBridge Test Page`、打印机名、生成时间。

校准页应使用矢量 PDF 绘制，避免先生成位图再转 PDF 带来的 DPI 和细线误差。

## 错误处理

接口返回明确错误，不提交不可验证的打印任务：

- 没有默认打印机：提示用户先选择默认打印机。
- 没有默认纸张：提示用户先选择默认纸张。
- 纸张尺寸非法：提示默认纸张尺寸无效。
- PDF 生成失败：提示测试页生成失败。
- 系统打印提交失败：沿用现有平台打印错误，例如打印机不存在或命令执行失败。

成功时返回“测试页已提交到系统打印队列”。

## 设置页设计

设置页的“打印设置”区域增加 `测试打印` 按钮。

按钮行为：

1. 如果当前设置存在未保存改动，提示用户先保存设置后再测试打印。
2. 调用 `POST /print/test`。
3. 成功后显示“测试页已提交到系统打印队列”。
4. 失败时显示后端返回的明确错误。
5. 调用结束后刷新日志列表。

设置页只新增这个操作入口，不改变现有打印机、纸张和端口配置流程。

## 托盘设计

托盘菜单把当前禁用的“测试打印（未实现）”改为可点击的“测试打印”。

托盘入口直接使用已保存配置执行测试打印。成功或失败都写入本地日志。托盘入口不弹出复杂窗口；用户需要看失败原因时，可以打开设置页或日志页查看。

## 测试策略

Rust 单元测试：

- 没有默认打印机时失败。
- 没有默认纸张时失败。
- 非法纸张尺寸失败。
- 生成的 PDF 是合法 PDF。
- 生成的 PDF 页面尺寸等于默认纸张尺寸。
- 成功时调用注入的 fake `PrintBackend`，参数包含正确打印机、纸张和份数 `1`。

HTTP 路由测试：

- `/print/test` 不再返回 `501 NOT_IMPLEMENTED`。
- 配置完整时返回成功。
- 配置缺失时返回明确错误。

前端验证：

- 有未保存设置时阻止测试打印并提示保存。
- 点击测试打印后展示成功或失败状态。
- 调用结束后能刷新日志列表。

平台手动验收：

- macOS 使用 CUPS `lp` 链路实测。
- Windows 使用 SumatraPDF 链路实测。
- 检查真实输出上的外边框、内边框、中心线和尺寸标记是否符合预期。
